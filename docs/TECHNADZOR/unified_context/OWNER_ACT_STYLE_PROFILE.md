## 6. Статусы замечаний (реальные формулировки из актов)

| Статус | Фраза из акта |
|---|---|
| Устранено | "ранее выявленные замечания ... подрядчиком исправлены" |
| Устранено частично | "устранено не в полном объёме" |
| Частично (с нюансом) | "основная часть ... устранена. Вместе с тем ..." |
| Не устранено | "данное замечание не устранено" |
| Требует доведения | "требует доведения до нормативного состояния ремонтным составом" |
| Новое замечание (не в предыдущих) | "Отдельно установлено, что ..." / "Отдельно выявлены ..." |
| Устно заявлено подрядчику | "Указанное замечание было заявлено исполнителю на объекте. Исполнитель сообщил, что данные участки будут устранены" |
| Принять к сведению | "Принять к сведению факт ..." |
| Без замечаний | "Замечаний по монтажу ... не выявлено" |

## 7. Язык и тон

- Технический, объективный, безличный
- Пассивные конструкции: "Установлено, что...", "Зафиксировано...", "Выявлено..."
- Без эмоций, без оценочных слов за пределами технических характеристик
- "вместе с тем" — для оговорки / частичного исполнения
- "в полном объёме" — стандартный маркер полноты устранения
- "нормативное состояние" — стандартное целевое состояние
- При хорошем результате: "выполнено качественно. Замечаний ... не выявлено. Отдельно фиксируется [нюанс], не влияющий на безопасность"

## 10. Раздел 3 — Рекомендовано к устранению

Нумерованный список. Каждый пункт — конкретное действие, глагол в инфинитиве:
```
1. Довести ... до нормативного состояния, устранить ...
2. Провести ... Дефектные участки зачистить, переварить ...
3. Выполнить ...
4. Организовать ...
```

## 13. Раздел 1 — Общие сведения

1-й выезд: цель осмотра — краткая (2-3 предложения).
3-й выезд (более зрелый акт): развёрнутый раздел с:
- кратким итогом по устранённым основным замечаниям
- перечнем выявленных новых недоделок
- дополнительными задачами текущего выезда
Это уместно при сложном объекте с историей.

## IMPORTANT: Что НЕ делать

- Не изобретать пункты нормативов (только номер документа или номер + пункт если известен)
- Не переносить дефекты с других объектов (Сусанино, Новичково)
- Не включать @tnz_msk как источник фактов объекта — только для улучшения структуры документа
- Не генерировать то, что "видно на фото" если Vision заблокирован
- Не указывать размеры/параметры если не подтверждены документацией или владельцем
- Если Vision заблокирован — добавить явную строку: "Автоматический визуальный анализ фото не выполнялся, так как Vision заблокирован. Выводы основаны на предыдущих актах, пояснениях владельца и доступных именах/метаданных файлов"

## Document Composition Methodology Summary

### Verified from @tnz_msk posts:

**Act structure:**
- Acts include: object address, date, participants, defect list, normative references, photo appendix, required actions
- Three-party acts (акт скрытых работ) require signatures from owner, contractor, and supervisor
- Post-check acts reference earlier defect list — каждый дефект либо устранён либо перенесён

**Defect wording:**
- Precise location required (axis, floor, apartment, zone)
- Measurement/scale where applicable (width, depth, area)
- Normative document cited (document number only unless source confirms section)
- Required action stated in imperative: «устранить», «выполнить», «предоставить»

**Photo-to-defect linking:**
- Each defect in act must reference photo number(s)
- Photos appended as numbered attachment or embedded in DOCX
- Photo caption must match defect location and type

**File workflow:**
- Act issued as DOCX (editable) + PDF (final signed)
- Inspector tools noted: прочность бетона, влагомер, тепловизор, ультразвук, georadar, нивелир
- XLSX for defect log: columns [№, описание, локация, норматив, фото, статус]

**Normative reference rule (QA-verified):**
- Use document number only: «СП 70.13330.2012», «ГОСТ 31937-2024»
- Do NOT add section/point unless the source explicitly states it
- If uncertain: write «норма не подтверждена» or «уточнить раздел»
